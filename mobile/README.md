# /mobile — Driver App (Flutter)

The on-the-road client where drivers post trips / return legs, see matched loads,
and manage bookings. Feature-first architecture with Riverpod, GoRouter, and Dio.

> **Status: scaffold only — not yet build-verified.** The environment that
> bootstrapped this repo had **no Flutter SDK**, so the Dart source and structure
> exist but `flutter pub get` / `flutter analyze` have **not** been run here.
> Platform folders (`android/`, `ios/`) are generated by `flutter create` (below).
> Full driver flows arrive with task **T-030**.

## Structure (feature-first)

```
lib/
├─ main.dart                 Entry point — wraps the app in a Riverpod ProviderScope.
├─ app.dart                  Root MaterialApp.router; reads router + theme.
├─ core/
│  └─ config/app_config.dart Compile-time config (API base URL via --dart-define).
├─ routing/app_router.dart   GoRouter provider (home route; more with T-030).
├─ services/dio_client.dart  Shared Dio HTTP client provider.
├─ shared/theme/app_theme.dart  Central Material 3 theme.
├─ widgets/section_card.dart Reusable UI building blocks.
└─ features/
   └─ home/home_screen.dart  Foundation landing screen.
```

## Chosen packages (see `pubspec.yaml`)

`flutter_riverpod` (state), `go_router` (routing), `dio` (HTTP),
`flutter_map` + `latlong2` (OpenStreetMap maps), `flutter_secure_storage`
(encrypted-at-rest tokens/cache — see [`../ai/OFFLINE_STRATEGY.md`](../ai/OFFLINE_STRATEGY.md)).

## Bring it to a runnable app

Requires the Flutter SDK (latest stable).

```bash
# From mobile/ — generate the platform folders into this existing package:
flutter create .
flutter pub get
flutter analyze
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8080/api/v1
```

`10.0.2.2` is the Android emulator's alias for the host machine, so it reaches the
API from [`../docker`](../docker) running on your laptop.
