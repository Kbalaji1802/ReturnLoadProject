# /docs — Human-Facing Documentation

Documentation meant for people: architecture explainers, API references, runbooks,
onboarding, and operational guides.

## How this differs from `/ai`

- **`/ai`** governs *how we build* — rules, business context, decisions, tasks. It
  is the operating system for contributors (human and AI).
- **`/docs`** explains *what we built and how to operate it* — architecture,
  API usage, and runbooks for anyone joining or running the system.

## Planned contents

- **architecture/** — system diagrams and deep-dives.
- **api/** — generated/curated API reference (from the OpenAPI contract).
- **runbooks/** — operational procedures (deploys, incidents, on-call).
- **onboarding/** — how a new engineer gets productive.
